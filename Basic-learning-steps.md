  The Best Way to Learn DSA Patterns

  Phase 1: Build the Foundation First

  Before patterns, solidify these basics:
  - Big-O notation — understand time/space complexity
  - Core data structures — Array, LinkedList, Stack, Queue, HashMap, Set, Tree, Graph, Heap
  - Know their operations, complexities, and when to use each

  ---
  Phase 3: The Learning Loop (Per Pattern)

  For each pattern, follow this cycle:
  1. Understand WHY this pattern exists (what class of problems does it solve?)
  2. Learn the template/skeleton code
  3. Solve 3-5 easy problems using ONLY that pattern
  4. Solve 3-5 medium problems
  5. Write down: "I reach for this pattern when I see ___"
  6. Revisit after 1 week (spaced repetition)

  ---
  Phase 4: Problem Recognition Training

  This is the real skill. After learning patterns, practice:
  - Reading a problem → identifying which pattern applies (before coding)
  - Platforms: LeetCode (pattern tags), NeetCode.io (best structured roadmap), Blind 75 → Grind 75 → NeetCode 150

  ---
  Key Principles

  - Never memorize solutions — understand the why behind every decision
  - One pattern at a time — don't context-switch between patterns
  - Struggle productively — 20-30 min stuck is fine, then look at hints (not full solutions)
  - Review wrong/slow solutions — this is where 80% of learning happens
